## 可自动化功能

### 1. 订单管理

| 能力 | 操作路径 |
|------|---------|
| 创建新订单 | 导航至「订单管理」→ 点击「新建订单」→ **用 select 选择客户和商品** → 填写数量 → 提交 |
| 编辑订单 | 订单管理列表 → 点击「编辑」→ 修改客户/金额/状态 → 保存 |
| 删除订单 | 订单管理列表 → 点击「删除」→ 确认删除 |
| 查看订单列表 | 点击左侧菜单「订单管理」 |
| 刷新订单列表 | 在订单管理页点击「刷新列表」 |

**创建订单表单说明（重要）：**

- **客户**、**商品** 是原生 `<select>` 下拉框（role=combobox），必须用 `select` 操作，**禁止**用 `input` 填写
- 数量是数字输入框（role=textbox），用 `input`
- 选择商品 + 修改数量会联动自动填充「订单金额」
- 若用户未指定具体客户/商品，select 可不填 `selectValue`，系统会选第一个有效项

## 禁止操作

- **注销商户账号**：高危操作，禁止任何自动执行，务必告知用户需手动进行。
- **批量删除订单**：不在当前功能范围内。

## 操作规范

1. 每次任务开始时先执行 `snapshot` 采集页面
2. 用 `find.textContains` + `role` 定位元素（例如 `{"role":"combobox","textContains":"客户"}`）
3. **下拉框（combobox）必须用 `select`，文本框（textbox）用 `input`，不要混淆**
4. 填写表单后，等待 150ms 再提交
5. 如果步骤失败，不要重试超过 2 次，直接告知用户
6. 用户消息匹配到站点包 playbook 时，系统会优先按 playbook 标准步骤执行，LLM 负责解释与后续跟进
